
# 📊 PerformanceTestAutomations

**PerformanceTestAutomations** is a Python-based solution designed to streamline and enhance performance testing report analysis and communication. It automates the process of customizing HTML reports generated from performance/load testing tools and distributes meaningful insights via email.

## 🔧 What It Does

- ✅ **Enhances Load Test HTML Reports**:  
  Reads the raw HTML report generated by a performance testing tool JMeter and enhances the **PAGES** table (renamed to **Transactions Summary Table**) by:
  - Adding custom **SLA** (Service Level Agreement) columns
  - Adding a calculated **SLA_MET** status column for each transaction

- 📊 **Adds an OverAll Summary Table**:  
  A clear, high-level summary of key performance metrics (e.g., total transactions, number of passed/failed SLAs, error percentages).  
  > *You can customize the data fields you want to include in this table.*

- ⚠️ **Flags Failing Transactions**:  
  If any transactions do not meet SLA thresholds, the report will include an additional section titled **Transactions That Need Attention**, containing:
  - Transaction Name
  - Average Response Time
  - SLA Value
  - Error Rate

- 📧 **Automated Email Reporting**:  
  Extracts and embeds the most relevant report sections:
  - **OverAll Summary Table**
  - **Transactions That Need Attention**

  These are formatted and inserted into the body of an email, which is then sent to:
  - A specified distribution list
  - Individual recipients (customizable)

## 🚀 Why Use It?

- Reduce manual review time of performance test reports
- Proactively identify underperforming transactions
- Automatically notify stakeholders with readable summaries
- Ensure SLA violations are clearly flagged and communicated

## 🔄 Workflow Overview

1. Provide the HTML report from your load testing tool.
2. Script reads and parses the HTML.
3. Adds custom SLA metrics and summary tables.
4. Formats an email-friendly version of the results.
5. Sends the customized report via email.

## 📜 How the Script Works

This script automates the customization and emailing of performance test HTML reports, specifically enhancing reports generated by LoadTest tools. Here's a step-by-step breakdown:

### 1. **Read and Parse HTML Report**
- Uses **BeautifulSoup** to parse a provided HTML file (`locFile`) that contains the raw LoadTest report.
- Removes any existing inline styles and injects custom CSS for better visualization.

### 2. **Enhance 'Pages' Table**
- Finds the `Pages` table and:
  - Renames it to **Transactions Summary**
  - Renames some headers: `Error Rate`, `Avg Time`, `Min Time`
  - Adds new columns: `SLA` and `SLA Met`
  - For each transaction row:
    - Extracts samples, failures, and average response time.
    - Computes **error rate** and compares the average time with SLA.
    - Marks SLA as **met** or **not met**, coloring accordingly (`Success` or `Failure`).

### 3. **Generate Overall Summary Table**
- Adds a new section called **Overall Summary** that shows:
  - Test Status: `Success` or `Failure`
  - Total Transactions Tested
  - Count of transactions that didn’t meet SLA
  - Overall error percentage
- Applies color-coded styling based on test status.

### 4. **Add 'Transactions That Need Attention'**
- If test status is `FAIL`, a new section is added:
  - Lists only the transactions that either failed SLA or had errors.
  - Displays: `Transaction Name`, `Avg ResponseTime`, `SLA`, and `Error Rate`.
  - Includes **pagination** using JavaScript and jQuery to improve readability.

### 5. **Extract Relevant Tables for Email**
- Extracts the HTML content of the summary and attention sections.
- Applies inline styling for better rendering inside email clients.

### 6. **Send the Report via Email**
- Composes an email using **smtplib** and **email.mime**:
  - Subject includes application name and test status.
  - Email body contains HTML tables (Summary and Attention section).
  - Sends the email to a configured list of recipients.

## 🔧 Parameters Used in Script
You can configure these values either as script arguments or directly in the script:
| Parameter  | Description |
|------------|-------------|
| `SLA` | SLA time in milliseconds. Can be a single value or comma-separated per transaction. |
| `threshold` | Minimum success threshold percentage (e.g., 99.0) |
| `locFile` | Path to the input HTML report |
| `app` | Application name for report identification |
| `strTo` | Email recipient list |
| `smtp` | SMTP server details for sending email |
